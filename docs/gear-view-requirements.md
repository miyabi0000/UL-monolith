# ギアビュー（カードビュー）機能 - 要件定義書

## 📋 概要

### 目的
現在のテーブル形式に加えて、画像を中心としたビジュアルなギア表示機能を提供し、ユーザーがギアの視覚的な確認・管理を容易にする。

### 背景
- 現在はテーブル形式のみでギアを管理している
- 画像（imageUrl）のデータは保存されているが、十分に活用されていない
- 視覚的にギアを確認したいニーズがある（特に装備選定時）

---

## 🎯 機能要件

### 1. 表示切替機能
- **ビュー切替ボタン**
  - テーブルビュー ⇄ カードビュー（ギアビュー）の切替
  - 切替状態はローカルストレージに保存
  - アイコン: 📋（テーブル）/ 🎴（カード）

### 2. カードビューのレイアウト
- **グリッドレイアウト**
  - レスポンシブグリッド（CSS Grid）
  - PC: 4カラム
  - タブレット: 3カラム
  - スマホ: 2カラム

### 3. カード表示内容

#### デフォルト状態（ホバーなし）
シンプルな表示で視認性を重視：
- **画像**
  - メイン画像（imageUrl）
  - No Image時のプレースホルダー
  - アスペクト比: 1:1（正方形）
  - 画像全体がカード領域を占める
- **カテゴリバッジ**（左上）: 小さなバッジ表示
- **不足警告**（右上）: 不足がある場合のみ⚠️アイコン

#### ホバー状態
カーソルを当てると、画像上にオーバーレイで詳細情報を表示：
- **半透明の暗いオーバーレイ**（背景）
- **ギア名**（name）: 大きく中央寄せ
- **ブランド名**（brand）: ギア名の下
- **詳細情報グリッド**:
  - 重量: `⚖️ 1.2kg`
  - 価格: `💰 ¥45,000`
  - 数量: `📦 1/1` (所有/必要)
  - 優先度: `⭐ High` (色付き)
- **アクションボタン群**（下部）:
  - 編集ボタン
  - 商品URLを開くボタン（productUrlがある場合）
  - 削除ボタン（小さく右下）

#### ホバーアニメーション
- カード全体が少し浮く効果（transform: translateY(-4px)）
- オーバーレイがフェードイン（opacity: 0 → 0.95）
- 情報が下からスライドイン
- トランジション: 200ms ease-out

### 4. カードアクション
各カードでできる操作：

#### マウスインタラクション
- **ホバー**: 
  - カード拡大 + 詳細情報オーバーレイ表示
  - アクションボタンの表示
- **クリック（カード全体）**: 
  - 編集フォーム表示
  - ホバー中のアクションボタンをクリックした場合は除外
- **ホバー中のアクションボタン**:
  - 📝 編集ボタン: 編集フォーム表示
  - 🔗 URLボタン: 商品ページを新しいタブで開く（productUrlがある場合のみ）
  - 🗑️ 削除ボタン: 削除確認後に削除

#### チェックボックスモード
- カード左上にチェックボックス表示
- ホバー状態でも常に見える位置

### 5. フィルタリング・ソート機能
テーブルビューと同じ機能を提供：
- **カテゴリフィルター**: 選択されたカテゴリのみ表示
- **ソート**: 
  - 名前（A-Z）
  - 重量（軽い順/重い順）
  - 価格（安い順/高い順）
  - 優先度（高い順/低い順）
  - 不足数（多い順）

### 6. 一括操作
- **チェックボックスモード**: カード左上にチェックボックス表示
- **一括選択**: すべて選択 / すべて解除
- **一括削除**: 選択したカードを一括削除

## 🎨 UI/UX要件

### デザインガイドライン
- **カードスタイル**: 
  - 白背景 + 軽いシャドウ（既存のCard UIコンポーネント使用）
  - ホバー時: 少し浮く効果（transform: translateY(-4px)）
  - 角丸: 8px（RADIUS_SCALE.md）
- **カラー**: 
  - 既存のdesignSystem.tsのCOLORSを使用
  - カテゴリカラーをアクセントに活用

### レスポンシブ対応
- **ブレークポイント**:
  - Mobile: < 640px (2カラム)
  - Tablet: 640px - 1024px (3カラム)
  - Desktop: > 1024px (4カラム)

### アクセシビリティ
- **キーボード操作**: Tab/Enter対応
- **スクリーンリーダー**: 適切なaria-label設定
- **代替テキスト**: 画像のalt属性設定

## 🔧 技術要件

### コンポーネント設計
```
client/components/
├── GearView.tsx          # カードビューのメインコンポーネント
├── GearCard.tsx          # 個別カードコンポーネント
└── ViewSwitcher.tsx      # テーブル⇄カード切替ボタン
```

### 状態管理
- **ビュー切替状態**: `useState` + `localStorage`
- **選択状態**: 既存のGearTableと同じロジック共有
- **ソート/フィルター**: 既存ロジックを再利用

### データフロー
```
App.tsx
  └─> [viewMode: 'table' | 'card']
      ├─> GearTable (viewMode === 'table')
      └─> GearView (viewMode === 'card')
```

### パフォーマンス最適化
- **画像遅延読み込み**: `loading="lazy"`
- **仮想スクロール**: 100件以上の場合のみ（react-window検討）
- **メモ化**: React.memoでカードコンポーネントをメモ化

## ✅ 受け入れ条件

### 必須要件
- [ ] テーブル⇄カード表示の切替ができる
- [ ] 画像付きカードでギアが表示される（デフォルト状態）
- [ ] 画像がない場合はプレースホルダーが表示される
- [ ] **カードホバー時に詳細情報オーバーレイが表示される**
- [ ] **ホバー時のアニメーション（浮く効果、フェードイン）が動作する**
- [ ] **ホバー中のアクションボタンが機能する**（編集、URL、削除）
- [ ] カードクリックで編集フォームが開く
- [ ] カテゴリバッジが左上に表示される
- [ ] 不足がある場合のみ警告アイコンが右上に表示される
- [ ] カテゴリフィルターが機能する
- [ ] レスポンシブレイアウト（2/3/4カラム）が機能する
- [ ] 既存のテーブルビュー機能に影響がない

### 推奨要件
- [ ] ソート機能が動作する
- [ ] 一括選択・削除機能
- [ ] 画像の遅延読み込み
- [ ] ビュー状態の永続化（localStorage）
- [ ] オーバーレイ情報のスライドインアニメーション
