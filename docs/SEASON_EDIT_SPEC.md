# Season機能とギア編集機能の改善仕様書

## 概要
GEAR LISTのテーブル表示において、Season（季節）機能をバー形式UIに変更し、ギアアイテムの編集UXをシンプル化する。

---

## 1. 現状の実装

### 1.1 TableRowの編集機能
- **現在**: 各行にインライン編集機能があり、Editボタンで編集モードに入る
- **編集可能項目**: Name（名前）、Seasons（季節）
- **Season UI**: ボタン形式のトグル（Spring/Summer/Fall/Winter）
- **問題点**:
  - インライン編集では全プロパティを編集しにくい
  - UXが煩雑

### 1.2 ActionMenuの機能
- **現在**: スリードットメニュー（⋮）から「Edit」「Save」「Delete」
- **位置**: 各行の右端（Actionsカラム → 現在はSeasonカラム）
-  Season機能実装でActionsカラムが削除

### 1.3 一括編集機能
- **BulkActionMenu**: チェックボックス表示のトグル
- **BulkActionModal**: 選択した複数アイテムの一括更新・削除
- **対応項目**: Category、Priority、Owned Quantity、Required Quantity

## 2. 改善仕様

### 2.1 Season UIの変更：スライダー形式（範囲選択）

#### 設計方針
Season表示を**スライダー式範囲選択バー**に変更し、視覚的に分かりやすく、直感的に季節範囲を指定できるUIにする。

#### 表示モード（View Mode）
```
┌─────────────────────────────────────────────────────┐
│ Spring    Summer    Fall      Winter                │
│ ●━━━━━━━━━━━━━━━━━━━━━━━━━●                        │
│ └─────── 選択範囲 ────────┘                         │
└─────────────────────────────────────────────────────┘
```

**仕様**:
- 水平スライダーで4つの季節を表示
- 選択された範囲は青色のバーで表示（bg-blue-500）
- 未選択部分は灰色（bg-gray-200 dark:bg-gray-700）
- 範囲のハンドル（●）で開始・終了季節を表示
- **編集不可**: 通常時はクリックやドラッグができない読み取り専用モード
- 選択なしの場合は「-」または空表示

**視覚例**:
- Spring-Summer: `●━━●────────────`
- Summer-Winter: `────●━━━━━━━━●`
- Fall のみ: `────────●●──────`
- 全季節: `●━━━━━━━━━━━━━●`

#### 編集モード（Edit Mode）
```
┌─────────────────────────────────────────────────────┐
│ Spring    Summer    Fall      Winter                │
│ ◉━━━━━━━━━━━━━━━━━━━━━━━━━◉  ← ドラッグで範囲調整  │
│ └─ 開始ハンドル    終了ハンドル ─┘                  │
└─────────────────────────────────────────────────────┘
```

**仕様**:
- **2つのハンドル**: 開始季節と終了季節を指定
- **ドラッグ操作**: ハンドルをドラッグして範囲を調整
- **連続範囲のみ**: Spring→Summer→Fall→Winterの順序で連続した範囲のみ選択可能
  - ✅ Spring-Summer-Fall（連続）
  - ❌ Spring + Winter（不連続、選択不可）
- ハンドルは大きめのサイズ（タッチ操作対応）
- ホバー時にハンドルが拡大（scale-110）
- ドラッグ中はカーソルが`cursor-grabbing`に変わる

### 2.2 編集機能の改善：スリードットメニュー活用

#### ActionMenuの復活と拡張

**位置**: Seasonカラムの右に新規カラム「Actions」を追加

**メニュー項目**:
```
⋮ （スリードットメニュー）
├─ Edit        → モーダルで全プロパティ編集
├─ Duplicate   → アイテムを複製（オプション）
└─ Delete      → 削除確認後、削除実行
```

#### 編集モーダル（Edit Modal）

**表示内容**: GearFormと同じフォームを表示し、すべてのプロパティを編集可能にする

**編集可能プロパティ**:
1. **Name** (名前): テキスト入力
2. **Brand** (ブランド): テキスト入力
3. **Category** (カテゴリ): ドロップダウン
4. **Product URL**: URL入力
5. **Image URL**: URL入力（プレビュー表示）
6. **Required Quantity** (必要数): 数値入力（1-10）
7. **Owned Quantity** (所有数): 数値入力（0-10）
8. **Weight (g)** (重量): 数値入力
9. **Price (¥)** (価格): 数値入力
10. **Season** (季節): バー形式UI（複数選択）
11. **Priority** (優先度): 1-5のドロップダウン

**ボタン配置**:
- **Save**: 変更を保存
- **Cancel**: 変更をキャンセルして閉じる
- **Delete**: 削除（モーダル内から削除も可能）

### 2.3 一括編集機能の拡張

#### BulkActionModalの拡張

**新規対応項目**:
- **Season** (季節): スライダー形式UIで範囲選択

**更新された項目リスト**:
1. Category（カテゴリ）
2. Priority（優先度）
3. Owned Quantity（所有数量）
4. Required Quantity（必要数量）
5. **Season（季節）** ← 新規追加

**UI仕様**:
```
┌────────────────────────────────────────────────┐
│  一括操作 (5個のアイテム)                      │
├────────────────────────────────────────────────┤
│  ○ 一括更新                                    │
│  ○ 一括削除                                    │
│                                                │
│  更新する項目: [Season ▼]                     │
│                                                │
│  Spring   Summer   Fall     Winter            │
│  ◉━━━━━━━━━━━━━━━━━━━◉  ← スライダーで範囲選択│
│                                                │
│  [キャンセル]  [更新実行]                     │
└────────────────────────────────────────────────┘
```

## 5. コンポーネント設計

### 5.1 新規コンポーネント

#### SeasonBar.tsx
```tsx
interface SeasonBarProps {
  seasons: string[]; // 選択された季節の配列（例: ['spring', 'summer', 'fall']）
  isEditing: boolean; // 編集モードかどうか
  onChange?: (seasons: string[]) => void; // 範囲変更時のコールバック
}
```

**責務**:
- 季節をスライダー形式で表示
- 編集モード時は2つのハンドルをドラッグして範囲を調整
- 連続した季節範囲のみ選択可能
- 読み取り専用モード時はドラッグ不可

**内部実装**:
- 季節の順序: `['spring', 'summer', 'fall', 'winter']`
- 選択範囲を開始インデックスと終了インデックスで管理
- ハンドルのドラッグイベントで範囲を更新
- 範囲から季節配列を生成: `seasons.slice(startIndex, endIndex + 1)`

#### EditGearModal.tsx
```tsx
interface EditGearModalProps {
  isOpen: boolean;
  gear: GearItemWithCalculated;
  categories: Category[];
  onClose: () => void;
  onSave: (updates: Partial<GearItem>) => void;
  onDelete: (id: string) => void;
}
```

**責務**:
- すべてのギアプロパティを編集可能
- GearFormのロジックを再利用
- モーダル内で保存・削除可能

### 5.2 更新コンポーネント

#### TableRow.tsx

- SeasonBarコンポーネントを使用（読み取り専用モード: `isEditing={false}`）

#### ActionMenu.tsx
- Editメニューで編集モーダルを表示

#### BulkActionModal.tsx

- Season一括更新機能を追加
- スライダーで範囲を指定して一括更新

## 6. データフロー

### 6.1 単一アイテム編集
```
TableRow (⋮ボタン)
  → ActionMenu (Editクリック)
    → EditGearModal (モーダル表示)
      → SeasonBar (Season編集)
      → onSave(updates)
        → API: PUT /api/v1/gear/:id
          → データベース更新
            → UIリフレッシュ
```

### 6.2 一括編集
```
BulkActionMenu (チェックボックス表示)
  → TableRow (複数選択)
    → BulkActionModal (一括操作モーダル)
      → Season選択 (SeasonBar使用)
        → onBulkUpdate({ seasons: [...] })
          → API: PUT /api/v1/gear (複数ID)
            → データベース一括更新
              → UIリフレッシュ
```

## 12. 想定される技術的課題

### 12.1 SeasonBarのレスポンシブ対応
- 小画面でスライダーが縮小される場合の対応
- 解決策: 最小幅を設定し、必要に応じてスクロール可能にする
- タッチデバイスでのハンドルサイズを大きめに設定（44x44px以上）

### 12.2 一括更新APIのパフォーマンス
- 多数のアイテムを一括更新する際の負荷
- 解決策: トランザクション処理とバッチ更新

### 12.3 状態管理の複雑化
- 編集モーダルとテーブルの状態同期
- 解決策: useCallbackで状態更新関数を安定化

### 12.4 ドラッグ操作の実装
- ハンドルのドラッグ中にマウス/タッチポインタがスライダー外に出る場合の処理
- 解決策: `document`レベルでイベントリスナーを登録し、グローバルな位置追跡
- クリーンアップを確実に実行（useEffectのreturn）

## まとめ

この仕様により、以下が実現されます：

1. **Season UIの改善**: スライダー形式で視覚的に分かりやすく、範囲選択が直感的
2. **読み取り専用表示**: テーブル上では常に編集不可、誤操作を防止
3. **編集UXのシンプル化**: スリードットメニュー→編集モーダルで全プロパティを編集
4. **一括編集の強化**: Seasonを含む複数プロパティの一括更新、スライダーで範囲指定
5. **テーブルのすっきり化**: インライン編集を削除し、見やすく

ユーザーは直感的に操作でき、誤操作なく効率的にギアアイテムを管理できるようになります。
