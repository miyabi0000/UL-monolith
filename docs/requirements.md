# UL Gear List Manager - 要件定義書

## 1. プロジェクト概要

### 1.1 目的
UL（ウルトラライト）志向のミニマリスト向けギア管理システム

### 1.2 対象ユーザー
- ウルトラライトハイカー
- ミニマリスト
- 軽量装備を重視するアウトドア愛好者

### 1.3 主要価値提案
- 商品URLからのLLM自動データ抽出
- 重量分析による円グラフ可視化
- 階層カテゴリによる柔軟な分類

## 2. 機能要件

### 2.1 コア機能
1. **ギアリストデータベース作成**
   - ギア情報の登録・編集・削除
   - 13項目のミニマルデータ管理

2. **グラフ生成**
   - システム別重量分析
   - 円グラフによる視覚化

3. **ギアリストプレビュー**
   - 印刷可能なリスト表示
   - 重量・価格集計

4. **ログイン機能**
   - ユーザー認証
   - 個人データ管理

### 2.2 データ項目（13項目）
| 項目 | 型 | 必須 | 説明 |
|------|-----|------|------|
| System | String | ○ | システムレイヤー |
| システムレイヤー | String | ○ | 日本語表記 |
| カテゴリー | String | ○ | 階層カテゴリ |
| 製品名 | String | ○ | 商品名 |
| 商品URL | String | - | 商品ページURL |
| ブランド | String | - | メーカー名 |
| 必要数 | Integer | ○ | 必要な数量 |
| 所持数 | Integer | ○ | 現在の所持数 |
| 不足数 | Integer | - | 計算値（必要数-所持数） |
| 単体重量g | Integer | - | 1個あたりの重量 |
| 総重量g | Integer | - | 計算値（単体重量×必要数） |
| 購入価格 | Integer | - | 価格（円） |
| 季節 | String | - | 使用季節 |
| 優先度 | Integer | - | 1-5の優先度 |

### 2.3 LLM自動抽出機能
- **入力**: 商品URL
- **抽出項目**: 製品名、ブランド、重量、価格、推定カテゴリ
- **確認フロー**: 抽出結果をユーザーが確認・修正可能

### 2.4 階層カテゴリシステム
- **構造**: System → システムレイヤー → カテゴリー → 製品
- **拡張性**: ユーザーが任意の階層を追加可能
- **集計**: 未マップカテゴリは「その他」として扱う
